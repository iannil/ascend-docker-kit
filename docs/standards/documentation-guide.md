# 文档规范指南

本指南定义 Ascend Docker Kit 项目的文档编写标准与规范。

## 1. 语言约定

- **交流与文档**：使用中文
- **代码注释**：使用英文
- **变量/函数命名**：使用英文

## 2. 命名规范

### 2.1 进度文档

格式：`YYYY-MM-DD-<功能描述>.md`

示例：

- `2026-01-18-project-initialization.md`
- `2026-01-20-cann-compatibility-matrix.md`

### 2.2 完成报告

格式：`YYYY-MM-DD-<功能描述>-report.md`

示例：

- `2026-01-18-project-initialization-report.md`
- `2026-01-25-core-modules-report.md`

## 3. 文档存放位置

| 文档类型 | 存放路径 | 说明 |
| --------- | --------- | ------ |
| 进行中的工作 | `docs/progress/` | 未完成的修改记录 |
| 完成报告 | `docs/reports/completed/` | 已完成工作的总结 |
| 验收报告 | `docs/reports/` | 对修改的验收记录 |
| 文档标准 | `docs/standards/` | 规范与指南 |
| 文档模板 | `docs/templates/` | 可复用的文档模板 |

## 4. 文档状态流转

```
[进行中] docs/progress/
    ↓ 完成后移动
[已完成] docs/reports/completed/
    ↓ 验收后创建
[验收报告] docs/reports/
```

## 5. 格式要求

### 5.1 必要元素

每个文档必须包含：

- **标题**：简明描述文档内容
- **日期**：文档创建或最后更新日期
- **状态**：进行中 / 已完成 / 已验收
- **概述**：简要说明文档目的

### 5.2 时间戳记录

修改过程中需要记录时间戳，便于追溯：

```markdown
## 修改记录

| 时间 | 内容 | 操作者 |
| ------ | ------ | -------- |
| 2026-01-18 14:30 | 创建初始文档 | - |
| 2026-01-18 16:00 | 添加模块说明 | - |
```

### 5.3 Markdown 规范

- 使用 ATX 风格标题（`#` 符号）
- 代码块指定语言类型
- 表格对齐
- 列表使用 `-` 符号

## 6. 文档维护原则

1. **及时更新**：每次修改后立即更新相关文档
2. **消除冗余**：删除重复或过期的内容
3. **保持一致**：遵循统一的格式与命名
4. **反映实际**：文档内容必须与代码/项目状态一致
